Приложение для управлением задачами

Функциональные требования
Каждый пользователь можте создавать таски. В таске он может указать title, description, audio-track.
Таску можно пометить как выполненную, можно удалить, можно редактировать. +
Пользователь должен иметь возможность изменить свой username
Должна быть админская часть где админ может наблюдать за следующими параметами
-Количество пользователей 
-Количетво тасок у пользователей
-Изменение количества пользователей за день\неделю\месяц
-Изменение количетва тасок за день\неделю\месяц
Еще нам нужно наблюдать какое количество запросов идет в наш API
Сколько ресурсов потребляет бэкэнд за минуту\неделю\месяц (pm2)
Планируется что приложение будет быстро развиваться и потребуется в последующем вредрение большого количество нового функционала.

Нефункциональные требовования
Authentiaction\ Authorization - сделать взаимозаменяемым (password js)
Swagger документация (Api dog)
Для фронта изспользовать React
Для типизации использовать Typescript
Библиоткеку компонентов хранить в Storybook
Бэк на NodeJS (REST)
Валидация запросов YUP
Ошибка как фронта так и бэка должны собираться в Sentry
Аналитику отправлять в Segment
Деплоймент должен быть в Kubernates
Мобайл фирст дизайн
Взаимодействие бэка и фронта через Axios
